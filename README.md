# 🪄 BG Remover

Удаление фона с картинок прямо в браузере. Фото никуда не загружаются — вся обработка идёт локально у пользователя (нейросеть @imgly/background-removal работает в браузере).

## Как выложить на GitHub Pages (5 минут)

1. Зайди на [github.com](https://github.com) → кнопка **New repository**.
2. Назови репозиторий, например `bg-remover`, видимость **Public**, жми **Create repository**.
3. На странице репозитория: **uploading an existing file** (или Add file → Upload files) → перетащи сюда все файлы из этого архива (`index.html`, `.nojekyll`, `README.md`) → **Commit changes**.
4. Открой **Settings → Pages** (в левом меню).
5. В блоке **Build and deployment**: Source — `Deploy from a branch`, Branch — `main`, папка — `/ (root)` → **Save**.
6. Через минуту-две сайт будет доступен по адресу:

```
https://<твой-логин>.github.io/bg-remover/
```

Эту ссылку и кидай кенту — работает у любого, ничего устанавливать не надо.

## Вариант через git (если привычнее консоль)

```bash
cd bg-remover-pages
git init
git add .
git commit -m "BG Remover"
git branch -M main
git remote add origin https://github.com/<твой-логин>/bg-remover.git
git push -u origin main
```

Дальше шаги 4–6 из инструкции выше.

## Если интернет медленный

На сайте есть выпадающий выбор модели:

| Модель | Размер | Когда выбирать |
| --- | --- | --- |
| ⚡ Лёгкая (isnet_fp16) | ~20 МБ | Медленный интернет, обычные фото — качество почти такое же |
| 💎 Нормальная (isnet) | ~40 МБ | Быстрый интернет, сложные кадры (волосы, мех, мелкие детали) |

Выбор запоминается в браузере. На 30 Мбит/с лёгкая модель качается ~5–7 секунд.

Также в сайт встроены **два CDN-зеркала** (jsdelivr и unpkg): если одно не открывается или тормозит у провайдера, библиотека автоматически берётся со второго.

## Важно знать

- **HTTPS обязателен** — GitHub Pages даёт его из коробки, так что всё ок.
- **Модель скачивается один раз** и кешируется браузером — следующие запуски мгновенные даже без быстрого интернета.
- **Приватность**: картинки не покидают браузер — на сервер ничего не уходит.
- Файл `.nojekyll` нужен, чтобы GitHub не прогонял сайт через Jekyll — не удаляй его.

## Функции сайта

- Выбор модели: лёгкая (~20 МБ) или качественная (~40 МБ) — выпадающий список над зоной загрузки
- Перетащи фото, выбери файл или вставь из буфера (Ctrl+V)
- Сравнение «до / после»
- Скачивание результата в PNG или WEBP (с автофолбэком на PNG, если браузер не умеет WEBP)
- Лимит размера файла 30 МБ, защита от двойного запуска
